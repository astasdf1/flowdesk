# Background Documents

This folder contains historical design and research material copied from earlier DEX Conductor planning work.

These files are **not normative** for FlowDesk implementation:

1. `OPENCODE_FIRST_PLUGIN_DESIGN.md`
2. `OPENCODE_PLUGIN_RESEARCH.md`

Use them only for context. If a background document conflicts with `../START_HERE.md`, `../USER_MANUAL.md`, `../FLOWDESK_OPENCODE_PLUGIN_IMPLEMENTATION_SPEC.md`, an ADR, the roadmap, the conformance plan, or the threat model, the normative FlowDesk document wins.

Legacy terms in this folder have these meanings:

1. DEX Conductor: historical predecessor context.
2. `@dex-conductor/*`: historical package examples, not FlowDesk package targets.
3. `.conductor/`: legacy or migration context, not the FlowDesk project data root.
4. `/flowdesk:*`: desired aliases only after conformance; portable `/flowdesk-*` commands are documented first for Release 1.
5. Command-first or developer-preview Release 1 wording: historical only. Current Release 1 is a chat-primary General-Use MVP with command-backed routing and fallback.

Do not implement production code directly from this folder.
