OMNIGENT × FLOWDESK

사내 에이전트 오케스트레이션,
한 번의 설치로 시작

Omnigent은 AI 에이전트를 실행·조율하는 플랫폼이고, FlowDesk는 작업마다 최적의 에이전트·모델·하네스를 골라 분배하는 오케스트레이터입니다. 이미 쓰는 Claude·Codex 구독을 그대로 활용합니다.

플랫폼 Omnigent 오케스트레이터 FD-OC 대상 macOS · Linux 소요 ~10분
01

무엇인가

두 조각을 함께 씁니다. 역할이 명확히 다릅니다.

Omnigent — 실행 플랫폼

로컬에서 서버 + 호스트를 띄우고 브라우저 UI로 세션을 만듭니다. 에이전트가 실제로 코드를 읽고, 쓰고, 터미널을 돌리는 런타임입니다.

FlowDesk — 선택·오케스트레이션 레이어

요청을 서브태스크로 쪼개, 각 작업에 맞는 에이전트·모델·하네스를 자문 선택해 분배하고 결과를 종합하는 오케스트레이터 에이전트(FD-OC)를 제공합니다. 라우팅만 하고 직접 구현하지 않습니다.

02

작동 방식

부모 오케스트레이터는 판단·분배만 하고, 실제 작업은 자식 세션이 각자의 하네스로 수행합니다.

요청

작업 입력

사용자가 워크스페이스 폴더를 고르고 할 일을 적습니다.

선택

FlowDesk 자문

작업 성격에 맞는 에이전트·모델·하네스를 고릅니다.

분배

서브태스크 실행

선택한 하네스로 자식 세션을 띄워 병렬 처리합니다.

종합

결과 합치기

결과를 모아 하나의 답으로 정리합니다.

구독 그대로 사용

부모 FD-OC는 Claude 구독(claude-sdk)으로 돌고, 서브태스크는 작업별로 네이티브 하네스(Claude → claude-native, OpenAI → codex, Gemini → antigravity-native)에서 실행됩니다. 별도 API 키 없이 로그인된 CLI 구독을 씁니다.

03

준비물

설치 전에 아래가 있으면 됩니다. 없으면 다음 단계에서 함께 설치합니다.

  • uv
    uv — Python 패키지/도구 매니저 · Python 3.12 포함 관리
  • tmux
    tmux — 네이티브 하네스(claude-native 등)가 터미널 세션에 필요 · brew install tmux
  • CLI
    Claude / Codex 로그인 — 구독 인증. claude 또는 codex로 로그인해 두면 그 구독을 사용
  • VPN
    Tailscale (선택) — 다른 기기에서 외부 접속할 때. 6번 섹션 참고
04

설치

두 번의 uv tool install로 끝납니다. 첫 명령은 Omnigent 안에 FlowDesk 선택 라이브러리까지 넣어 가드/선택이 동작하게 하고, 두 번째는 런처·MCP 실행 파일을 PATH에 올립니다.

1

Omnigent + FlowDesk 선택 라이브러리

Omnigent 본체를 설치하고, 같은 환경에 FlowDesk 선택 라이브러리를 포함(--with)합니다.

2

FlowDesk 런처 · 도구

flowdesk-omnigent-start, -stop, -install-templates, -mcp 실행 파일을 설치합니다.

3

FD-OC 오케스트레이터 템플릿

FD-OC · FD-OC-Opus · FD-OC-Codex를 ~/.omnigent/agents에 설치합니다.

4

프로바이더 로그인 (구독)

쓰는 것만 로그인하면 됩니다. 로그인된 구독을 그대로 사용합니다.

05

첫 실행 (로컬)

서버와 호스트를 한 명령으로 띄우고 브라우저를 엽니다.

  1. 왼쪽 위 New session → 작업할 폴더(워크스페이스) 선택
  2. 에이전트에서 FD-OC(또는 FD-OC-Opus / FD-OC-Codex) 선택
  3. 할 일을 적으면 FlowDesk가 서브태스크로 분배 → 결과를 종합합니다
자율 실행

FD-OC는 서브태스크를 확인 없이 분배하고 결과를 기다립니다. 중간에 불필요한 질문을 하지 않고, 실패가 실제로 발생했을 때만 짧게 보고합니다.

06

외부 접속 — Tailscale

다른 노트북·폰에서 접속하려면 Tailscale로 tailnet을 만들고, 로컬 서버를 tailnet 안에서만 HTTPS로 노출합니다. 인터넷 전체 공개가 아니라 팀 tailnet 안에서만 보입니다.

1

Tailscale 설치 · 로그인

서버를 돌릴 이 컴퓨터에 설치하고, 회사 tailnet에 로그인합니다.

2

Omnigent를 --tailscale로 기동

--tailscale는 이 노드의 MagicDNS 주소·Tailscale IP를 자동으로 신뢰 목록에 넣어, 외부 브라우저에서 새 세션 생성이 막히지 않게 합니다.

3

tailnet에 HTTPS로 노출

로컬 127.0.0.1:6767을 tailnet 안에서 HTTPS로 프록시합니다. 한 번 설정하면 유지됩니다.

4

다른 기기에서 접속

접속할 기기에도 Tailscale을 설치하고 같은 회사 tailnet에 로그인한 뒤, 위에서 확인한 주소를 브라우저에 엽니다.

접근 권한 = Tailscale

로컬 단일 사용자 모드에는 별도 로그인이 없어, tailnet에서 이 서버에 닿을 수 있는 사람은 누구나 사용할 수 있습니다. “누가 쓸 수 있는가”는 회사 Tailscale 초대·ACL로 관리하세요. 인터넷 공개 tailscale funnel은 쓰지 않습니다.

07

중지 · 관리

런처가 백그라운드에 있거나 터미널을 닫았어도 한 명령으로 정리합니다.

08

자주 겪는 것

정상입니다. FlowDesk 오케스트레이터는 FD-OC · FD-OC-Opus · FD-OC-Codex 셋입니다. 나머지 *-native-ui 카드(Claude Code, Codex 등)는 Omnigent 기본 제공 항목입니다.
외부 브라우저에서 새 세션이 안 만들어져요 (403)403
Omnigent의 CSRF 보호가 loopback 외 브라우저 Origin을 막기 때문입니다. flowdesk-omnigent-start --tailscale로 기동하면 이 노드의 ts.net 주소가 자동 허용됩니다. 바인딩 주소(--bind)를 바꾸는 것과는 무관합니다.
서브태스크는 어떤 하네스로 도나요?harness
부모 하네스와 무관하게 작업별로 선택됩니다. Claude 계열은 claude-native(네이티브·구독), OpenAI는 codex, Gemini는 antigravity-native. 부모가 claude-sdk여도 서브태스크는 네이티브로 실행됩니다.
flowdesk-omnigent-start를 찾을 수 없대요PATH
uv tool update-shell을 실행하고 새 터미널을 열어 ~/.local/bin이 PATH에 있는지 확인하세요.
네이티브 세션이 안 떠요tmux
claude-native 등 네이티브 하네스는 tmux가 필요합니다. brew install tmux (또는 apt install tmux) 후 다시 시도하세요.
