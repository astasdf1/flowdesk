On 2026-06-02 the event-based watchdog transition design was iterated to `docs/STALL_RECOVERY_PLAN_V11.md` V11.2 after two more rounds of multi-model critical review (Claude Opus, GPT, Gemini). The confirmed philosophy is: EVENTS are the primary source of truth for WHEN a child session is done; event EXCEPTIONS (missing/delayed/out-of-order events) are handled by EXPLICIT TIMEOUTS placed behind a tool-running guard (NOT by silence-as-idle inference, which was the root cause of the slice1c mid-turn `MessageAbortedError` and the zero-event `no_response` bugs); polling is demoted to body-text extraction only. The second review round surfaced three "termination-guarantee holes" that V11.2 closes: G1 `unknownStateMaxMs` (a dropped tool-settle event demotes the lane to `toolStateUnknown`, which blocks abort/continuation and is force-terminated only after the unknown dwell exceeds the cap, preventing a permanent limbo), G2 `absoluteLaneAgeMs` (a true zero-event lane still terminates via an absolute age cap independent of the meaningful-activity silence gate), and G3 expected-turn binding (capture binds to the expected assistant message rather than "latest", with no latest fallback) plus G4 non-circular idle dwell, G5 bounded `awaiting_body_capture`, and G6 schema minimization. Slice ordering was reworked so the tool-set + tool-gated timeout terminator lands first (Slice 1) and the event capture fast-path layers on top.
