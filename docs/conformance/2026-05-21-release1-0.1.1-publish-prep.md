# Release 1 0.1.1 Publish Prep

Date: 2026-05-21

## Scope

This note records local publish-preparation evidence for `@flowdesk/opencode-plugin@0.1.1`. It does not publish to npm, mutate the active OpenCode profile, call providers, run `opencode run`, enable real dispatch, launch lanes, or prove hard chat cancellation/no-reply authority.

## Version Change

`@flowdesk/opencode-plugin` was bumped from `0.1.0` to `0.1.1` to carry the exact `@opencode-ai/plugin@1.15.6` dependency pin for OpenCode 1.15 compatibility. `@flowdesk/core` remains `0.1.0` because no core package source changed for this prep step.

## Publication Gate

Publishing was explicitly approved after local prep, but npm rejected the publish attempts with `EOTP` and required one-time-password or browser CLI authentication. Retrying with `NODE_AUTH_TOKEN` sourced from the local npm token file still returned `EOTP`, so the available token is not sufficient for this publish path or the account/package requires a current OTP/web-auth completion.

Attempted command:

```sh
npm publish --workspace @flowdesk/opencode-plugin --access public
```

Publication remains blocked until an OTP, a publish-capable automation token, or completed npm CLI web authentication is available. After publish, FlowDesk must run a fresh public-registry install smoke and active-profile migration smoke before claiming that `0.1.1` is the public Release 1 OpenCode 1.15 package path.
