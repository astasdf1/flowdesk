# Release 1 0.1.1 Publish Completion

Date: 2026-05-21

## Scope

This note records the public npm publication of `@flowdesk/opencode-plugin@0.1.1`, the post-publish fresh public-registry install smoke, and the active OpenCode profile migration from `@flowdesk/opencode-plugin@0.1.0` to `@flowdesk/opencode-plugin@0.1.1`. It did not call providers, run `opencode run`, enable real dispatch, launch lanes, switch providers/models, prove hard chat cancellation/no-reply authority, or claim any later-gated capability.

## Publication

`@flowdesk/opencode-plugin@0.1.1` was published to the public npm registry under the `flowdesk` org with public access. Direct `npm publish` and `NODE_AUTH_TOKEN`-based attempts both returned `EOTP`; publication succeeded only after writing the local npm token into a temporary `.npmrc` and passing it through `--userconfig`. The temporary `.npmrc` was deleted immediately after publish, and the token file contents were never printed.

Working command sequence:

```sh
NPMRC=/var/folders/yh/84gs38c94211rknpy09lpz_40000gn/T/opencode/flowdesk-publish-npmrc
umask 077
{ printf '//registry.npmjs.org/:_authToken='; tr -d '\n\r' < "$HOME/Documents/work/projects/npm_tk.txt"; printf '\n'; printf 'registry=https://registry.npmjs.org/\n'; } > "$NPMRC"
chmod 600 "$NPMRC"
npm whoami --userconfig="$NPMRC"
npm publish --workspace @flowdesk/opencode-plugin --access public --userconfig="$NPMRC"
rm -f "$NPMRC"
```

Registry verification:

- `npm view @flowdesk/opencode-plugin version` returned `0.1.1`.
- `npm view @flowdesk/opencode-plugin dist-tags` returned `{ "latest": "0.1.1" }`.
- `npm view @flowdesk/opencode-plugin@0.1.1 dependencies` returned `{ "@flowdesk/core": "^0.1.0", "@opencode-ai/plugin": "1.15.6" }`.

`@flowdesk/core` was not re-published; the published version remains `0.1.0` because no core source changed for this release.

## Interpretation

This closes the OpenCode 1.15 Release 1 package-distribution blocker. The published `@flowdesk/opencode-plugin@0.1.1` resolves to `@opencode-ai/plugin@1.15.6` and `@opencode-ai/sdk@1.15.6`, fresh public-registry install smoke passes, and the active OpenCode profile loads the new plugin with 11 tools, `chat.message` steering, and all real-dispatch/lane-launch/fallback/hard cancel authority flags false.

It does not promote any later-gated capability. Real provider dispatch, actual OpenCode subtask/model/provider lane launch, automatic fallback/reselection, production model selection, hard chat cancellation/no-reply authority, and top-tier reviewer lane orchestration remain blocked until their separate conformance and Guard gates pass.
