# OpenCode 1.15 Compatibility Check

Date: 2026-05-21

## Scope

This check records the first compatibility pass after the local OpenCode CLI updated from the prior `1.14.50` evidence line to the `1.15.x` line. It does not call providers, run `opencode run`, enable real dispatch, launch lanes, switch providers/models, or prove hard chat cancellation/no-reply authority.

## Environment

- Local OpenCode CLI: `1.15.5` from `opencode --version`.
- npm registry latest observed during the check:
  - `opencode-ai@1.15.6`
  - `@opencode-ai/plugin@1.15.6`
  - `@opencode-ai/sdk@1.15.6` via `@opencode-ai/plugin`.
- `npm view opencode version` returned npm `E404`; the package line used by this environment is `opencode-ai`.

## Dependency Update

The workspace plugin dependency was updated from `@opencode-ai/plugin@1.3.12` to exact `@opencode-ai/plugin@1.15.6`. Exact pinning is intentional because FlowDesk conformance evidence is tied to concrete OpenCode package surfaces and should not float silently to newer plugin typings.

`@opencode-ai/plugin@1.15.6` brings `@opencode-ai/sdk@1.15.6` and the newer Effect-based plugin/event dependency surface into the lockfile.

## Verification

Completed locally:

- `npm install @opencode-ai/plugin@1.15.6 --workspace packages/opencode-plugin`
- `npm run typecheck`
- `npm test` passed with 277/277 tests.
- Direct module load of `packages/opencode-plugin/dist/server.js` returned plugin id `flowdesk`, 11 tools, and `chat.message` present.

The TypeScript build accepted the existing FlowDesk plugin code against `@opencode-ai/plugin@1.15.6` without code changes.

## Compatibility Interpretation

No immediate TypeScript-level API break was found for the Release 1 plugin surface:

- `Plugin`, `PluginModule`, `PluginOptions`, and `tool` imports still compile.
- Release 1 tool registration remains type-compatible.
- The existing visible `chat.message` steering hook remains source-compatible in FlowDesk code.
- The opt-in managed-dispatch adapter remains an injected-client boundary and is not enabled by default.

This check does not prove runtime behavior for OpenCode `1.15.x`. Release 1 remains command-backed and non-dispatch by default. Real provider dispatch, actual delegated lane launch, managed fallback/reselection, hard chat cancellation/no-reply authority, and top-tier reviewer lane orchestration remain later-gated until separate runtime conformance proves the updated OpenCode surface.

## Follow-Up Conformance Needed

Before promoting any additional OpenCode `1.15.x` authority, FlowDesk still needs pinned runtime evidence for:

1. Plugin load and command/chat smoke in an isolated profile using `@opencode-ai/plugin@1.15.6`.
2. FDS-1 schema conversion and runtime-closed unknown-property rejection under the updated plugin package.
3. `chat.message` mutation, throw behavior, duplicate assistant behavior, and lack or presence of first-class no-reply/cancel authority.
4. SDK `session.prompt`, `promptAsync`, `children`, `messages`, provider/model echo, event delivery, telemetry correlation, and abort behavior.
5. Auth/provider metadata and sanitizer assumptions against the updated OpenCode/auth-plugin environment.
